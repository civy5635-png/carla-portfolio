    Project 2: Interactive Banner

    View Interactive Banner Project

    I decided to do an interactive banner for a software called 'Missed Call Text Back'. It uses AI to immediately text a potential customer for a business if the business misses the call, keeping them engaged and preventing them from simply moving on to the following number on the list. My ultimate goal is to build a business that helps small service businesses improve their online presence and attract more customers by offering services such as AI employees, business websites, and reputation management. I'm learning how to set this up now, while I'm taking this class. I want this web design class to get me closer to my goals. A year ago, I didn't know how to connect my printer to my computer or even create a PDF file, so I've actually come a long way.

    I clicked the banner on the homescreen and used the pre-set dimensions of 468 X 60. According to ChatGPT, this is kinda old school, and advised me to use a size of 300 × 250, 728 × 90, or similar, but I already started with 468 × 60, so I decided to stick with that, and possibly do a second version if it doesn't take the whole 6 hours(perhaps wishful thinking). I set it for 12 FPS, and started with #3a193f to color the stage, but ChatGPT suggested creating a background layer with its own timeline would be smarter, so I did. From Canva, I downloaded a graphic of a ringing cell phone and an edited Text box for the banner. I also added a business logo, a copy from my own website. My idea was to animate a ringing cell phone, moving side to side, with the text 'Missed Calls = Missed Customers. When the mouse or finger hovers, it changes to an image of my logo, the edited text box, and the text 'We Text Them Back'. I created separate layers for the ringing cell phone, the missed call text, my logo, text box, 'we text them back' text, and the background layer, turning all of them into symbols. On ChatGPT's suggestion, I created a 3-frame animation of the cell phone moving back and forth using classic tween, set it to loop, and placed it on an actions layer. I started by trying to use code snippets in the action panel, but I ran into issues. I never used this successfully in the class exercises, and this time was no different. In class, the only way I got the exercises to work was by pasting code ChatGPT wrote for me in the actions panel. The first few attempts caused everything to run at once when I used Ctrl+Enter to preview. So that was day one.

    The next day, I spent 5 hours trying to fix the code, making sure everything was on its own layer and not attached to anything else, and all the instance names were correct, but every time, everything happened all at once. I count on ChatGPT to write code because I only know a little about that. I concluded that either I wasn't giving it the right or enough information, or I was making some code-breaking mistake that was sabotaging me. Also, ChatGPT could be hallucinating-it's sent me down many time sucking rabbit holes when it acted like it thoroughly knew what was wrong, and it very much didn't. I kept at it because I wanted to see why it wasn't working-never did find out. At the 6th hour, after hitting my keyboard with my head, I asked it for a different way to do this, and it suggested doing it as a button. I deleted everything and started over. I'd forgotten about buttons. I created one layer with the background, turned it into a symbol/button, double-clicked it, and saw the button frames. On the first frame, I put everything I wanted on the default frame (phone_ringing, text graphic), cleared it, and added my hover symbol (logo, text box, and outgoing text) to the 2nd and 3rd frames, then added just the background to the 4th frame. The first time I tested it, the phone moved, and the text flashed. I was still using the motion tween on the phone. I deleted the motion tween, had ChatGPT write the code to make the phone shake, and added it to the action panel in the phone symbol. I retested it (Ctrl+Enter), and it worked! It took about 45 minutes. I was torn between celebrating and fantasizing about taking a chainsaw to ChatGPT for wasting 5 hours of my life. I do have to concede that the button section was toward the end of the class, and it was my own fault for forgetting about it. I considered adding a phone ringing sound file, but it was incredibly annoying, so I removed it.

    This was less frustrating than the character animation, but only by a day. I did learn a lot, though. I wish I'd done the banner before I did the character animation(project 1). If I had, the character animation would have been a lot better. I would have known I needed to size the stage to match the background so the movie wouldn't be off-center with all that extra stage showing. When I started, I clicked 'Banner' on the home page and kept all the default settings, except for the FPS (changed to 12 FPS). I haven't been very clear about the different kinds of files and how they differ. When I went to publish, guided by ChatGPT, I found that Flash Player was defunct and that I should have started this on an HTML Canvas. I'm adding it to my export file as my first version, but I'm also creating a 2nd version in HTML Canvas, copying all the assets into the 2nd version's library. I created a layer(purple_BG_button) and turned it into a button symbol. I then double-clicked it, bringing up the button frames. In the first frame, I double-clicked the phone (phone_mc) to create a 4-frame animation that made it move back and forth within the symbol. I backed out of the phone symbol, locked the first frame, added the hover_view to the next two frames, then added just the background to the 4th frame. I tested it (Ctrl+Enter), and it looked good, so I exported the file and added it to my Project Two folder. This was definitely less frustrating than the character animation.
